🎯 我們要做什麼？

  1. 情境：常去全聯買王子麵跟香皂。出門前（或結帳前）想知道大概要付多少錢—— 身上的錢夠不夠？
  2. 手上有的：過去九筆購買紀錄——每次買了幾包麵、幾個香皂、付了多少錢。

為什麼要深度？— 線性的極限與神經元的誕生

王子麵的新規則：單價 10 元，超過 20 包整單打 8 折 （20 包 200 元，22 包反而只要 176 元）。我們手上只有偶數包數的購買紀錄 （都是湊雙數買的），老闆問：那買 奇數包會是多少錢？ 資料不再是一條直線——線性模型組合來組合去還是線性的，造不出「轉折」。 解法：給模型加上會轉折的神經元（ReLU）。 而且這一次，我們用模型沒看過的奇數包來驗收它學得好不好。

整個深度學習，其實只有三個動作 （全部都是國高中的幾何概念）

🔄 W（權重）＝旋轉 調 w 是改變斜率——把直線（高維時是超平面）在空間中旋轉， 轉到最貼合資料的角度。下面的互動就有一顆 w 旋鈕，轉轉看。
↕️ b（偏置）＝平移 調 b 是改變截距——把整條線在空間中上下平移， 移到資料坐落的位置。王子麵 tab ① 的「底費 c」就是它。
🌀 非線性函式＝擠壓／彎折 ReLU 把線折彎、tanh 把整條數線擠壓進 −1~1。 沒有它，疊一百層還是一條直線；有了它，才能貼合彎曲的世界。

深度學習密合資料，靠的就是這三個動作的層層堆疊： 旋轉（W）、平移（b）、擠壓（非線性）—— Yann LeCun（深度學習三巨頭之一）在紐約大學的課堂上就是這樣教的。 學習深度學習真的不難：它的零件，你在國高中都學過了。

完全吻合，就是最好嗎？— Overfitting（過度擬合）

注意 14 顆練到極致的那一幕：曲線在訓練點之間凹凹凸凸地亂長， 15 個實心點全都貼得漂漂亮亮（訓練誤差 ≈ 0），但空心的奇數包全部被亂報—— 測試誤差衝到一百多。它背下了 15 筆紀錄，卻沒有學到「打折規則」， 這就是 overfitting。而且過程有一個關鍵訊號——在 14 顆模式按自動訓練時， 盯著左邊的「誤差走勢」小圖：訓練誤差（藍）一路下降， 測試誤差（紅）掉到谷底後反轉上揚。兩條線分岔的那一刻， 就是模型從「學規則」滑向「背資料」的瞬間（實測：訓練誤差 38→3 全程下降， 測試誤差中途觸底 41 之後一路爬回 110）。三種模式排在一起就是機器學習的三幕劇：

模式訓練誤差測試誤差診斷
📏 純線性≈ 184≈ 180太簡單，學不會（underfit）
🧠 2 顆（放對位置）≈ 0≈ 0剛剛好，學到規則
🌀 14 顆（練到極致）≈ 0> 100背答案，沒學規則（overfit）

所以：訓練誤差愈低，不等於愈好——測試誤差（沒看過的資料）才是裁判。 再按「⚡ 資料加點雜訊」讓對比更兇：14 顆會連雜訊都背起來、測試誤差依然爆炸； 2 顆反而因為「背不動」，畫出平滑的線、更接近真實規則。 14 顆的表達力本身沒有錯——錯在參數多、資料少、又練到極致； 真實的深度學習用海量資料與正則化技巧來馴服它。

🤔 兩個思考題 （先自己想，再點開提示）

1. 為什麼這個 case 剛好是「2 顆神經元」有最佳結果？

因為老闆的規則本身就是兩段直線的銜接——需要的「彎折」剛好是兩個。 2 顆神經元提供的形狀自由度跟真實規則同形： 不少（足以做出必要的轉折）、也不多（沒有多餘的彎可以拿去背雜訊）。 所以「最適合的神經元數量」取決於真實規則有多複雜—— 規則愈彎曲，需要的神經元愈多。 但真實世界通常不會告訴你規則長什麼樣（人臉有幾個「彎」？）， 沒辦法像這裡一樣算出「剛好幾顆」——所以我們才需要下一題的裁判。

2. 所以，我們重視的是訓練誤差，還是測試誤差？

測試誤差。回想 tab ① 的初衷：訓練是為了推論—— 對「沒看過的資料」做出正確預測（出門前算出要帶多少錢）。 訓練誤差只是我們拿來調旋鈕的線索，不是目的； 模型的真本事永遠用它沒看過的資料來驗收。 這就是為什麼機器學習實務上一定把資料切成訓練集／測試集 （tab ③④ 的 72 張測試圖、這一頁的奇數包，都是這樣來的）， 也是為什麼剛剛的走勢圖上，我們盯的是紅線、不是藍線。

一顆神經元的解剖

神經元 = 加權和（線性：旋轉＋平移）＋ 啟動函數（非線性：擠壓／彎折）。 ReLU 超簡單：負的變 0、正的照過——把線折彎； tanh 則是把任何數擠壓進 −1~1 之間（tab ③ 的網路用的就是它）。 三個動作在這一顆小東西裡到齊了。

深度 = 模型自己找特徵

用「身高、體重、學歷」直接預測「交友運」很難——中間隔了太多層因果。 網路的中間層會自己長出「健康、有前途、有名氣、很活躍」這類中間特徵， 先墊一層，再用這層特徵往上推。 問題更複雜時，一層特徵不夠用，就需要特徵的特徵—— 再墊一層、再墊一層，這就是為什麼需要「更多層」、為什麼叫「深度」學習。 關鍵是分工：人只決定要幾層、每層幾顆神經元（跟決定輸入輸出一樣，是設計）； 每個特徵長什麼樣，是訓練自己找出來的——人沒教過它「健康」是什麼， 它長出來的特徵我們也未必看得懂，但誤差會告訴它找得好不好。 （tab ③ 的第一層權重熱圖，就是網路自己長出來的特徵偵測器。）

黑盒子的信念

模型的方程式可以跟真實世界的規則長得完全不一樣——我們不再要求「看懂它」， 只要求一件事：誤差調得小，就是好盒子。 海量神經元、海量資料、海量計算，全部靠觀察誤差變化來調整參數，就會有效果。 需要一點想像力、需要接受這個信念——這就是 Scaling Law（規模法則）的基礎， 也是 GPT 用 1750 億顆旋鈕能學會語言的原因。

兩層機制的分工 （差在「看的範圍」）

    

      

        
word embedding：全語料的身分證

        

          讀完所有句子，統計「誰常跟誰一起出現」，每個詞得到一個固定向量。
          常出現在相似上下文的詞（男星／女星、台灣／珍珠奶茶）向量就相近——
          訓練完成後，這張身分證就不變了。

        🗺️ 全局・靜態——一詞一向量，走到哪帶到哪
      

      

        
attention：當下句子的關係

        

          處理句子時，每個詞去「看」句內其他詞，算出該注意誰、注意多少。
          同一個「蘋果」，在「蘋果很好吃」和
          「蘋果推出新手機」裡注意的對象完全不同——
          詞的「當下意義」由句子決定。

        🔦 局部・動態——同一個詞，每句注意的對象都不同
      

    

    
更精確地說：真正的 transformer 裡 embedding 不是先訓練好才拿來用，
      而是跟整個模型一起訓練；attention 看的也不只一句話，而是整段上下文，並且用學出來的
      Q／K／V 投影與多頭機制。但「全局字典 → 當下關係」這個分工，就是它的核心直覺。

word2vec — 詞向量是「訓練」出來的

    
tab ⑥ 的詞地圖是統計算出來的；業界更經典的做法是 word2vec：
       訓練一個只有一層 hidden layer 的神經網路玩「猜鄰居」遊戲（skip-gram），
       訓練完把輸出層丟掉，把輸入層的權重直接當作詞向量。
       這一頁用同一份 66 句語料，在你的瀏覽器裡現場訓練一次給你看。

  


  

    
網路長什麼樣？ （一層 hidden layer，輸入層權重就是詞向量）

    
    
輸入是 one-hot：整排只有「男星」那一格是 1，其他都是 0——
      所以第一層權重矩陣（102×4）中只有「男星」那一列會被選中、被更新。
      訓練完，那一列就是「男星」的 4 維詞向量。Keras 的 Embedding 層做的正是這件事。

    

      

        
🐱🐶 貓狗論證（用我們語料的「男星／女星」）

        

          語料裡「男星 與 女星 爆出 婚訊」「南韓 女星 與 男星 爆出 緋聞」——
          兩個詞的鄰居幾乎相同，也就是兩筆訓練資料想要相同的輸出。
          要讓輸出相同，最省力的辦法就是把兩列權重調成幾乎一樣——
          梯度下降就這樣「被逼著」把同類詞的向量推在一起。

      

      

        
📊 跟 tab ⑥ 殊途同歸

        

          tab ⑥ 用統計（共現＋降維）、這一頁用訓練（梯度下降），
          做出來的地圖性質相同——理論上 skip-gram 等價於對 PMI 矩陣做隱式分解。
          差別是 word2vec 只需一對一對地看資料，語料再大都訓練得動，
          這是它成為業界標準的原因。

      只是這裡的「旋鈕」是第一層的 102×4 = 408 個權重，而且訓練完不丟掉——
      拿來當每個詞的身分證。（實測發現只給 2 維會讓 102 個詞塞得太擠、方向糊在一起，

"""迷你 word2vec（skip-gram）：單一 hidden layer 的網路，取輸入層權重當詞向量。

全 softmax cross-entropy + SGD。編碼維度 D=4，訓練完的 W1 每一列
就是一個詞的 4 維詞向量（D=2 時 102 個詞塞進 2 維會塌縮、失去區分度，

WIN, D, LR, EPOCHS, SEED = 2, 4, 0.05, 60, 1

sents = [line.split() for line in CORPUS.strip().split("\n")]

idx = {w: i for i, w in enumerate(vocab)}
V = len(vocab)

# skip-gram 訓練對：(中心詞, 視窗內的鄰居)
pairs = np.array([
    (idx[s[i]], idx[s[j]])
    for s in sents
    for i in range(len(s))
    for j in range(max(0, i - WIN), min(len(s), i + WIN + 1))
    if j != i
])
print(f"詞彙量 V={V}，訓練對 {len(pairs)} 筆")

rng = np.random.default_rng(SEED)
W1 = rng.normal(0, 0.5, (V, D))   # 輸入層權重：訓練完就是詞向量！
W2 = rng.normal(0, 0.5, (D, V))   # 輸出層權重：訓練完就丟掉

for epoch in range(EPOCHS):
    rng.shuffle(pairs)
    loss = 0.0
    for c, o in pairs:                    # c=中心詞、o=要猜的鄰居
        h = W1[c]                         # one-hot 選中的那一列（前向傳播）
        z = h @ W2
        z -= z.max()
        p = np.exp(z)
        p /= p.sum()                      # softmax：對每個詞的預測機率
        loss += -np.log(p[o] + 1e-12)     # cross-entropy
        p[o] -= 1.0                       # 反向傳播（dz）
        dW1 = p @ W2.T
        W2 -= LR * np.outer(h, p)
        W1[c] -= LR * dW1                 # 只有被選中的那一列會更新
    if epoch % 10 == 0 or epoch == EPOCHS - 1:
        print(f"epoch {epoch:3d}  平均 loss = {loss / len(pairs):.4f}")

def cos(a, b):
    va, vb = W1[idx[a]], W1[idx[b]]
    return va @ vb / (np.linalg.norm(va) * np.linalg.norm(vb) + 1e-12)

def neighbors(w, k=4):
    v = W1[idx[w]]
    sims = W1 @ v / (np.linalg.norm(W1, axis=1) * np.linalg.norm(v) + 1e-12)
    order = np.argsort(-sims)
    return [(vocab[i], round(float(sims[i]), 2)) for i in order[1:k + 1]]

print("\n=== 貓狗實驗（上下文相同的詞，權重被訓練得幾乎一樣）===")
print(f"cos(男星, 女星)     = {cos('男星', '女星'):+.2f}   ← 應接近 +1")
print(f"cos(熱戀, 緋聞)     = {cos('熱戀', '緋聞'):+.2f}   ← 應接近 +1")
print(f"cos(男星, 珍珠奶茶) = {cos('男星', '珍珠奶茶'):+.2f}   ← 不相關，應該低")

print("\n=== 最近鄰 ===")
for w in ("男星", "蘋果", "GPT", "台灣"):
    print(f"{w} → {neighbors(w)}")

print("\n=== PCA 投影成 2D（網頁版地圖用同樣的方法畫散佈圖）===")
Wc = W1 - W1.mean(axis=0)
U, S, Vt = np.linalg.svd(Wc, full_matrices=False)
coords2d = U[:, :2] * S[:2]
for w in ("男星", "女星", "蘋果", "台灣"):
    x, y = coords2d[idx[w]]
    print(f"{w}: ({x:+.2f}, {y:+.2f})")



  







  

    
鸚鵡上學記 — 從 GPT 到 ChatGPT

    
tab ⑤ 訓練出來的 GPT 是一隻全世界最有見識的鸚鵡：1750 億個參數、
       看過 3000 億個字詞，超會接龍——但牠只會「接話」，不會「對話」，
       常常弄不清楚人的意圖、常常說錯話。
       從 GPT 到 ChatGPT，就是送這隻鸚鵡去上學的過程。

  


  

    
上學三部曲

    

      

        
🦜

        起點
        
GPT：很會接龍的鸚鵡

        
用整個網路的文字玩接龍（自監督），學會了世界知識與語言——
           但只會順著你的話往下接。

      

      

        
👩‍🏫

        SFT
        
找老師示範什麼是對話

        
人類寫好幾萬筆「怎麼問、就該怎麼答」的示範對話餵給牠——
           注意：這是人給答案的監督式學習，不是自監督。

      

      

        
🏆

        RM
        
參加比賽，讓大家評分

        
讓模型對同一題寫好幾個回答，人類幫回答排名。
           用這些排名訓練一個「評分模型」（Reward Model），學會人類的品味。

      

      

        
🎓

        RLHF
        
照評審的品味自我修煉

        
之後不用人盯著：模型自己產生回答、RM 打分數、
           模型朝高分方向調整參數（強化學習）——畢業，成為 ChatGPT。

      

    

  


  

    
「接話」跟「對話」差在哪？

    

      

        
🦜 GPT（上學前）只會接話

        

          你說：「請幫我規劃一個三天兩夜的台北旅遊行程」

          牠接：「的注意事項有哪些？以下整理十點……」——
          牠只是在猜「這句話後面最可能出現什麼字」，不是在回答你。

      

      

        
🎓 ChatGPT（上學後）會對話

        

          同一句話，牠回：「好的，Day 1：台北 101、信義區；Day 2：故宮、
          士林夜市；Day 3：陽明山、北投溫泉。」——
          因為老師（SFT 的人類示範）教過牠：這種句子是「請求」，要給「回答」。

      

    

  


  

    
換你來：你就是人類評審 （這就是 RM 的訓練資料是怎麼來的）

6 筆成對比較

從這次經驗學到的 vibe coding 心法

    

      
💡 說「目的」，不是「規格」
        第一輪 prompt 講的是教學目標（讓學生體會梯度下降），而不是「幫我畫 20 個 canvas 圓圈」。
        AI 理解目的後，很多細節它會自己補齊、甚至想得比你周到。

      
❓ 讓 AI 發問，關鍵決策自己拍板
        好的 AI 協作不是一次把需求寫完美，而是允許它反問。
        預訓練方式、旋鈕形式、bias 處理——三個問題三個選擇，就避開了三次重工。

      
🧪 要求「驗證過」而不是「寫完了」
        Claude Code 會自己開瀏覽器操作頁面：拖旋鈕、按按鈕、對照理論值（歸零時誤差 = ln 2 ≈ 0.693）。
        寫完 ≠ 能用，驗證過才算數。

      
🔁 小步迭代，一次只加一件事
        先讓核心功能能動（旋鈕挑戰）→ 再重排版面（加 tab）→ 再加說明頁（這一頁）。
        每一步都建立在「上一步已驗證 OK」的基礎上。
